# CIP Repository Repair, Integration, Verification & Production-Readiness Plan

## Executive Summary

CIP (Constituency Intelligence Platform) is a government constituency management system with a Python/FastAPI backend, Next.js 16 frontend, PostgreSQL/PostGIS database, and Redis. The codebase has a well-designed Clean Architecture but has **critical security issues, missing dependencies, broken authentication wiring, and no frontend-backend contract enforcement**. This plan takes the repo from its current state to a verified, deployable system.

---

## Initial State Assessment

| Area | Current State | Severity |
|------|--------------|----------|
| **Secrets** | Hardcoded JWT secret key in `authentication.py` line 9; hardcoded DB credentials in `settings.py` line 18 | CRITICAL |
| **Dependencies** | `PyJWT` and `passlib[bcrypt]` missing from `requirements.txt` — Docker build will fail | CRITICAL |
| **Auth** | Auth decorator exists but is never applied to any route — all endpoints fully public | CRITICAL |
| **SQL Injection** | f-string interpolation in `my_issues.py:27` and `postgis.py:19` | HIGH |
| **Health/Readiness** | Health always returns 200; readiness only checks DB, not Redis | MEDIUM |
| **Frontend/Backend Sync** | No OpenAPI spec generation; no contract enforcement; officer pages import from `mp` namespace | HIGH |
| **Tests** | Only 1 test file (`test_system.py`, 4 tests); no frontend tests | HIGH |
| **Docker** | No `.dockerignore`; backend Dockerfile will fail due to missing deps | HIGH |
| **CI** | No frontend CI; backend CI installs deps separately (not from requirements.txt) | MEDIUM |
| **Frontend** | All pages client-rendered; no middleware.ts; no error boundaries; no shared components; login always redirects to `/citizen` | MEDIUM |
| **Rate Limiting** | In-memory only, never applied to routes | LOW |
| **Workers** | Defined but never started in app lifecycle | LOW |
| **Stubs** | `AssessSubmissionWorkflow`, `EnrichIssueWorkflow`, priorities endpoint, chat history — all return empty/hardcoded data | INFO (document) |
